MASTERING NOSQL

Le Big Data et l'IoT ont redéfini les règles. Les bases de données traditionnelles ont atteint leurs limites. Plongez dans l'architecture distribuée qui propulse l'intelligence artificielle et le Cloud moderne.

DÉCRYPTER LES 7 CHAPITRES

01 // Définition & Comparaison

NoSQL (Not Only SQL) est une approche flexible qui s'affranchit du modèle relationnel rigide pour exceller dans la gestion massive de données non structurées.

L'Ancien Monde (Relationnel / SQL)

  • Structure : Tables rigides avec schémas fixes.
  • Scalabilité Verticale : Augmentation du CPU/RAM d'un seul serveur (très coûteux).
  • Transactions : ACID strict, cohérence forte immédiate.
  • Cas d'usage : Applications transactionnelles critiques (banques).

Le Nouveau Monde (NoSQL)

  • Structure : Schémas flexibles et documents imbriqués.
  • Scalabilité Horizontale : Ajout de serveurs standards en cluster.
  • Transactions : BASE, acceptant une "cohérence éventuelle".
  • Cas d'usage : Big Data, Temps réel, Réseaux sociaux.

Les 4 Grandes Familles :

1. Clé-Valeur (ex: Redis)

Chaque clé unique est associée à une valeur. Idéal pour le cache et les sessions.

2. Documents (ex: MongoDB)

Stockage JSON/BSON semi-structuré. Parfait pour les applications Web et CMS.

3. Colonnes (ex: Cassandra)

Données organisées par colonnes. Maître de l'analytique Big Data et IoT.

4. Graphes (ex: Neo4j)

Modélisation par nœuds et relations. Expert pour les réseaux sociaux.

04 // L'ADN NoSQL : Le Format JSON

Le JSON (JavaScript Object Notation) est la base du stockage dans MongoDB. Il est lisible pour l'humain et très rapide à parser pour les machines.

  • Objets { } : Collections de paires clé-valeur.
  • Tableaux [ ] : Listes ordonnées de valeurs.
  • Imbrication : Un document peut en contenir d'autres à l'infini.
{
  "nom": "Guarraz",
  "prenom": "Mouad",
  "age": 20,
  "competences": [ "JavaScript", "React", "MongoDB" ],
  "adresse": {
    "ville": "Rabat",
    "pays": "Maroc"
  }
}

06 // Stratégie des Liens

A. Embedding (Imbrication)

Intégration complète des données liées dans un seul document (ex: Le client possède directement ses adresses dans un tableau [ ]).

  • ✅ Vitesse Extrême : Pas de jointure, 1 seule requête suffit.
  • ✅ Usage : Relations 1:1 et 1:N lues ensemble.

B. Referencing (Références)

Stockage d'identifiants (ObjectIDs) pointant vers d'autres documents (ex: Une commande qui contient l'ID du client).

  • ✅ Intégrité : Évite la duplication, mises à jour faciles.
  • ✅ Usage : Relations N:M ou données lourdes.

07 // Architecture Interne MongoDB

1

Base de Données

Le conteneur logique de plus haut niveau qui regroupe plusieurs collections (ex: ecommerce).

2

Collections

Groupes de documents similaires (ex: products).

3

Documents

Enregistrements individuels JSON/BSON avec un identifiant unique obligatoire _id.

Le Namespace : C'est la combinaison unique `BaseDeDonnées.Collection`.
Exemple : ecommerce.products.
