MASTERING NOSQL

NoSQL désigne une nouvelle génération de bases conçues pour le Big Data, assurant scalabilité horizontale et performances extrêmes.

CONSULTER LE DOSSIER

01 //Choc des Architectures

[Image of NoSQL horizontal scalability vs SQL vertical scalability]

Relationnel (SQL)

  • Structure : Tables rigides avec schémas fixes.
  • Scalabilité : Verticale (augmentation de puissance).
  • Transactions : ACID strict, cohérence forte.

NoSQL

  • Structure : Schémas flexibles et documents imbriqués.
  • Scalabilité : Horizontale (ajout de serveurs).
  • Transactions : BASE, cohérence éventuelle.

02 //JSON & Métamorphose

[Image of MongoDB BSON document structure vs SQL table row]

Le JSON est léger et parfait pour MongoDB. Dans ce nouveau monde, la modélisation change radicalement :

  • Tables → Collections.
  • Lignes → Documents JSON.
  • Jointures → Imbrication (Embedding).
{
  "_id": ObjectId("65eef..."),
  "nom": "Dupont",
  "competences": ["NoSQL", "React"],
  "adresse": { "ville": "Paris" } // Imbrication
}
